Inheritence and Cascade

CSS stands for Cascading Style Sheets, and the first word cascading is stupidly complicated to understand yet at the same time is pretty simple.

Stylesheets cascade â€” at a very simple level, this means that the origin, the cascade layer, and the order of CSS rules matter. When two rules from the same cascade layer apply and both have equal specificity, the one that is defined last in the stylesheet is the one that will be used.

MDN Docs

Inheritence is also kind of weird. Some properties are inherited from parent elements while others are not. For example, color and font-family are inherited values, but width is not. You can control inheritence by setting a property value to something like inherit, which turns on inheritence. initial sets the value to the initial value of the CSS specification. revert sets the value to the browser default. unset acts like revert, except that if the property usually inherits it will inherit. Finally, revert-layer, resets the property value to one defined in a previous cascade layer (which is its own whole thing).

The property value all lets you use the above inheritence values be applied to all properties at once!
